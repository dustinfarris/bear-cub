# Out-of-scope discoveries land here during implementation instead of being fixed inline. Promoted to v2 candidates at initiative close.

- =points_total/2= (=lib/bear_cub/chores.ex=, Story 02) issues one query for distinct =(routine, local_date)= pairs, then one =routine_day_contribution/3= call — two more queries — per pair, an N+1 shape. Correctness is unaffected and all tests pass, but the total is cumulative all-time and recomputed on every =:chores_changed= re-fetch (Story 03 wires the kiosk badge to it), so this could become a real cost as history grows. Candidate for =docs/backlog.org= if it shows up as a perf concern once the badge is live.
- A pre-existing set of =KioskLiveTest=/=Admin.TodayLiveTest=/=Admin.ChoreLiveTest= tests (predating Story 06) call the real clock (=auto_routine()= or equivalent) without pinning routine windows, so they fail whenever the suite happens to run inside the 23:00–05:00 Good Night gap — confirmed via a =git stash= baseline at the same wall-clock time (same 15 test names fail identically with or without Story 06's changes, so this is not a Story 06 regression). Reconfirmed identically during Story 07 (same wall-clock hour). Candidate for =docs/backlog.org=: adopt the =morning_active=/=put_windows= pinning helpers (already used by Story 06's tests and the "collapse band" describe block) across the older, unpinned tests.
- =BearCubWeb.KioskLive='s collapse-delay (=lib/bear_cub_web/live/kiosk_live.ex=, Story 07): a rapid undo/redo of a routine's last chore within the =@collapse_delay_ms= (400ms) window can schedule more than one =Process.send_after(self(), {:collapse_ready, kid_id}, ...)= for the same kid — the first timer's reference is never stored or cancelled on undo. The extra message is a harmless no-op (=MapSet.delete= on an already-absent key, then an idempotent reload), so no incorrect UI state results, but a very fast undo→redo could make the band appear slightly earlier than a full fresh delay after the final completion. Not covered by any AC — the design's "short delay" is explicitly imprecise at this level. Candidate for =docs/backlog.org= if it ever surfaces as an on-device feel issue.
